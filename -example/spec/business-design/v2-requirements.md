# V2.0 业务需求 - 模板系统、导入导出、工作流编排

## 新增用户故事

### US7: 模板管理
作为用户，我可以创建提示词模板，支持变量占位符。

**验收标准：**
- 可以创建模板，包含标题、内容、类型
- 模板内容支持变量占位符语法：`{{变量名}}`
- 可以定义变量元数据（名称、类型、默认值、描述）
- 可以从模板实例化提示词，自动填充变量
- 可以编辑/删除模板

**变量类型支持：**
- text: 文本输入
- number: 数字输入
- select: 下拉选择（需定义选项）
- boolean: 布尔值

**示例：**
```
模板内容：生成一张{{style}}风格的{{subject}}图片，光线为{{lighting}}
变量定义：
  - style: type=select, options=[写实, 卡通, 油画, 水彩], default=写实
  - subject: type=text, required=true
  - lighting: type=text, default=自然光
```

### US8: 批量导入导出
作为用户，我可以批量导入导出提示词。

**验收标准：**
- 支持导出为 JSON/CSV/Markdown 格式
- 支持从 JSON/CSV 导入提示词
- 导出时可选择导出范围（全部/选中分类/选中标签）
- 导入时自动创建不存在的分类、标签、模型
- 支持导出模板
- 提供导出模板供用户参考

**导出格式示例：**

JSON:
```json
{
  "version": "2.0",
  "exported_at": "2026-03-20T10:00:00Z",
  "prompts": [...],
  "categories": [...],
  "tags": [...],
  "models": [...]
}
```

Markdown:
```markdown
# 提示词：美丽的日落

**类型**: image  
**分类**: 自然风光  
**标签**: 风景, 日落  
**模型**: DALL-E 3  

## 内容
一张美丽的日落照片，金色阳光洒在海面上...
```

### US9: 工作流编排
作为用户，我可以创建多步骤工作流，组合多个提示词。

**验收标准：**
- 可以创建工作流，定义执行步骤
- 每个步骤关联一个提示词或模板
- 支持步骤间的变量传递（上一步输出 → 下一步输入）
- 可以定义步骤执行顺序（顺序/并行）
- 可以执行工作流并记录执行历史
- 支持查看工作流执行日志

**工作流结构：**
```
工作流：生成博客文章配图
├── 步骤1: 生成文章大纲 (模板)
│   └── 输出: outline
├── 步骤2: 生成配图提示词 (模板)
│   ├── 输入: outline (来自步骤1)
│   └── 输出: image_prompt
└── 步骤3: 生成配图 (提示词)
    └── 输入: image_prompt (来自步骤2)
```

**步骤类型：**
- prompt: 执行指定提示词
- template: 基于模板生成并执行
- transform: 数据转换（格式化、提取等）

**执行状态：**
- pending: 待执行
- running: 执行中
- completed: 已完成
- failed: 失败
- skipped: 跳过
