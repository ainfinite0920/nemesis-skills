## 2026-03-20-V2-模板系统导入导出工作流

### 目的

为提示词管理系统添加模板系统、批量导入导出、工作流编排功能。

### 范围

**新增实体：**
- Template: 提示词模板，支持变量占位符
- Workflow: 多步骤工作流编排
- WorkflowExecution: 工作流执行记录

**新增功能：**
- 模板管理 CRUD
- 模板实例化（变量填充）
- 批量导入导出（JSON/CSV/Markdown）
- 工作流管理 CRUD
- 工作流执行引擎
- 执行历史记录

### 预期行为

1. 用户可以创建带变量的模板，从模板快速生成提示词
2. 用户可以批量导入导出提示词和模板
3. 用户可以创建多步骤工作流，组合多个提示词/模板
4. 工作流执行时支持变量传递和状态追踪

### 新增 API 端点

**模板 API：**
- `GET /api/templates` - 获取模板列表
- `GET /api/templates/{id}` - 获取模板详情
- `POST /api/templates` - 创建模板
- `PUT /api/templates/{id}` - 更新模板
- `DELETE /api/templates/{id}` - 删除模板
- `POST /api/templates/{id}/instantiate` - 实例化模板
- `POST /api/templates/{id}/validate` - 验证模板

**工作流 API：**
- `GET /api/workflows` - 获取工作流列表
- `GET /api/workflows/{id}` - 获取工作流详情
- `POST /api/workflows` - 创建工作流
- `PUT /api/workflows/{id}` - 更新工作流
- `DELETE /api/workflows/{id}` - 删除工作流
- `POST /api/workflows/{id}/execute` - 执行工作流
- `GET /api/workflows/{id}/executions` - 获取执行历史
- `GET /api/executions/{id}` - 获取执行详情
- `POST /api/executions/{id}/cancel` - 取消执行

**导入导出 API：**
- `GET /api/export?format=json|csv|markdown` - 导出
- `POST /api/import` - 导入

### 数据库变更

新增表：
- `templates` - 模板表
- `workflows` - 工作流表
- `workflow_executions` - 执行记录表
